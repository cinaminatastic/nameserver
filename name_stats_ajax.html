Software Development - Shakespeare Lookup Program

This program is designed to demonstrate a simple client/server program. A server program runs in the background, waiting for Word/Data requests. When it receives a word request it returns the lines containing the word.

The reason for such a architecture is to allow a simple program, with a MAP preloaded with data, to quickly provide results without incurring the overhead of a reloading the Shakespeare data for each request.
(Written by Alex Seaver and Jim Skon)
